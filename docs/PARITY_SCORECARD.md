# Parity scorecards (legacy)

Strict pydantable **2.0** is defined by the **typed dataframe spec** and the strict runtime
contract (typed column tokens + explicit `*_as` schema evolution).

Older “parity scorecard” documents tracked 1.x feature parity against other dataframe
libraries and facade surfaces. Those parity documents are **not part of the strict 2.0
contract**, and the pandas/PySpark facade surfaces are removed in strict 2.0.

- **Strict 2.0 spec**: {doc}`typed_dataframe_spec`
- **Migration**: {doc}`MIGRATION_1_to_2`
